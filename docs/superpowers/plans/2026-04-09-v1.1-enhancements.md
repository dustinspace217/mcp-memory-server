# mcp-memory-server v1.1 Enhancements — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add 9 enhancements inspired by mempalace (point-in-time queries, importance scoring, context layers, memory type classification, observation metadata management, wake-up summary, duplicate check, audit log, stats) that improve memory recall quality and drift resistance across compactions and session restarts. No breaking changes.

**Architecture:** The codebase has 6 source files (`types.ts`, `cursor.ts`, `embedding.ts`, `jsonl-store.ts`, `sqlite-store.ts`, `index.ts`). This plan adds 1 new file (`audit.ts`) and modifies all existing files except `embedding.ts`. Schema v5 to v6 migration adds three columns: `importance`, `context_layer`, and `memory_type`. All other features use existing schema or are additive tools.

**Tech Stack:** TypeScript, Node.js 22, better-sqlite3, sqlite-vec, @huggingface/transformers, Zod, Vitest, MCP SDK

**Design Spec:** `docs/superpowers/specs/2026-04-09-v1.1-enhancements-design.md`

```typescript
// importance: REAL 1.0-5.0 score (default 3.0 = medium).
//   Used by get_summary and get_context_layers to prioritize observations.
// context_layer: TEXT nullable. NULL = L2 (on-demand, default for all existing data).
//   'L0' = always loaded (~100 token budget, core identity/rules).
//   'L1' = loaded at session start (~800 token budget, active work/decisions).
//   Inspired by mempalace's L0-L3 layered memory system.
// memory_type: TEXT nullable. NULL = unclassified.
//   Free-form tag classifying the nature of the observation (not the entity).
//   Recommended values: 'decision', 'preference', 'fact', 'problem', 'milestone', 'emotional'.
//   Enables filtering queries by type (e.g., "show me recent decisions").
if (currentVersion < 6) {
  this.db.exec(`ALTER TABLE observations ADD COLUMN importance REAL NOT NULL DEFAULT 3.0`);
  this.db.exec(`ALTER TABLE observations ADD COLUMN context_layer TEXT DEFAULT NULL`);
  this.db.exec(`ALTER TABLE observations ADD COLUMN memory_type TEXT DEFAULT NULL`);
  this.db.prepare('UPDATE schema_version SET version = 6').run();
  currentVersion = 6;
}
```

Add a helper that builds the temporal WHERE clause:

```typescript
// Builds SQL conditions for point-in-time observation filtering.
// When asOf is provided, returns only observations that were active at that moment:
// created before asOf AND either still active or superseded after asOf.
private temporalObsFilter(asOf?: string): { clause: string; params: string[] } {
  if (!asOf) return { clause: `superseded_at = ''`, params: [] };
  return {
    clause: `created_at <= ? AND (superseded_at = '' OR superseded_at > ?)`,
    params: [asOf, asOf]
  };
}

// Same pattern for relations.
private temporalRelFilter(asOf?: string): { clause: string; params: string[] } {
  if (!asOf) return { clause: `superseded_at = ''`, params: [] };
  return {
    clause: `created_at <= ? AND (superseded_at = '' OR superseded_at > ?)`,
    params: [asOf, asOf]
  };
}
```

- [ ] **Step 1: Create audit.ts**

```typescript
import * as fs from 'node:fs/promises';

/**
 * Append-only JSONL audit logger for knowledge graph mutations.
 * Each log entry records the operation name, a summary of the input
 * (not full content -- just names and counts), and the outcome.
 *
 * Fire-and-forget from MCP handlers -- failures are logged to stderr
 * but never propagate to callers.
 */
export class AuditLogger {
  // Path to the .audit.jsonl file, derived from the main database path
  private logPath: string;
  // Whether logging is enabled (disabled by MEMORY_AUDIT_LOG=off)
  private enabled: boolean;

  constructor(dbPath: string, enabled: boolean = true) {
    this.logPath = dbPath + '.audit.jsonl';
    this.enabled = enabled;
  }

  /**
   * Appends a single audit entry to the log file.
   * Creates the file on first write. Safe for single-writer (this process).
   *
   * @param op - The operation name (e.g., 'create_entities', 'add_observations')
   * @param input - Summary of the operation input (names, counts -- not full content)
   * @param result - Summary of the outcome (counts, success/failure)
   */
  async log(
    op: string,
    input: Record<string, unknown>,
    result: Record<string, unknown>
  ): Promise<void> {
    if (!this.enabled) return;
    try {
      const entry = JSON.stringify({
        ts: new Date().toISOString(),
        op,
        input,
        result,
      });
      await fs.appendFile(this.logPath, entry + '\n');
    } catch (err) {
      // Audit logging must never fail the operation -- log to stderr and continue
      console.error(`Audit log write failed: ${err}`);
    }
  }
}
```

```typescript
// Token budget enforcement: L0 ~100 tokens (~400 chars), L1 ~800 tokens (~3200 chars)
const L0_CHAR_BUDGET = 400;
const L1_CHAR_BUDGET = 3200;

let l0CharCount = 0;
const warnings: string[] = [];

// L0: include all but warn if over budget (never truncate L0 -- it's core rules)
for (const obs of l0Results) {
  l0CharCount += obs.content.length + obs.entityName.length + 5;
}
if (l0CharCount > L0_CHAR_BUDGET) {
  warnings.push(`L0 exceeds 100-token budget (est: ${Math.ceil(l0CharCount / 4)} tokens)`);
}

// L1: include until budget exhausted, sorted by importance DESC
let l1CharCount = 0;
const l1Included: typeof l1Results = [];
for (const obs of l1Results) {
  const cost = obs.content.length + obs.entityName.length + 5;
  if (l1CharCount + cost > L1_CHAR_BUDGET) break;
  l1Included.push(obs);
  l1CharCount += cost;
}

const totalChars = l0CharCount + l1CharCount;
return {
  L0: l0Results,
  L1: l1Included,
  tokenEstimate: Math.ceil(totalChars / 4),
  warnings
};
```

```typescript
setObservationMetadata(updates: SetObservationMetadataInput[]): Promise<number> {
  let totalUpdated = 0;
  const txn = this.db.transaction(() => {
    for (const u of updates) {
      // Look up entity
      const entity = this.db.prepare('SELECT id FROM entities WHERE name = ?').get(u.entityName);
      if (!entity) throw new Error(`Entity not found: ${u.entityName}`);

      // Build SET clause dynamically based on which fields are provided
      const sets: string[] = [];
      const params: unknown[] = [];
      if (u.importance !== undefined) { sets.push('importance = ?'); params.push(u.importance); }
      if ('contextLayer' in u) { sets.push('context_layer = ?'); params.push(u.contextLayer); }
      if ('memoryType' in u) { sets.push('memory_type = ?'); params.push(u.memoryType); }
      if (sets.length === 0) continue;

      params.push(entity.id, u.content);
      const result = this.db.prepare(
        `UPDATE observations SET ${sets.join(', ')} WHERE entity_id = ? AND content = ? AND superseded_at = ''`
      ).run(...params);
      totalUpdated += result.changes;

      // Update entity's updated_at if any observation was modified
      if (result.changes > 0) {
        this.db.prepare('UPDATE entities SET updated_at = ? WHERE id = ?')
          .run(new Date().toISOString(), entity.id);
      }
    }
  });
  txn();
  return Promise.resolve(totalUpdated);
}
```

## Future / v2.0 candidates

| Feature | What it solves | Why deferred |
|---------|---------------|-------------|
| **"What changed since T"** | After session break: "what happened while I was away?" A `get_changes_since(timestamp)` returning created/superseded/invalidated items. | Read-only query against existing columns. Simple, but plan is full. |
| **Entity aliasing / merge** | "mcp-memory-server" and "memory server" fragment observations across duplicates. `merge_entities(source, target)` consolidates. | Needs schema design (alias table or redirect), cascading relation updates. |

| **Contradiction detection** | Two active observations state conflicting facts. Similarity check catches near-duplicates but not semantic contradictions. | Needs semantic comparison, possibly LLM-in-the-loop. Mempalace has this but hasn't integrated it either. |
